04
Configurer l'IA ✨ IA

GameDay utilise l'API Anthropic (Claude) pour deux fonctionnalités : la génération automatique du programme et l'estimation des durées. Ces fonctionnalités sont optionnelles — le site fonctionne entièrement sans.

Fonctions utilisant l'IA
✨ Génération du programme
Génère automatiquement un programme de journée optimisé selon les votes, les contraintes de joueurs et les teachers. Utilise Claude Sonnet (mode libre) ou Claude Haiku (mode guidé).
⏱ Estimation des durées
Estime automatiquement la durée d'un créneau en fonction du jeu, du nombre de joueurs et du teaching. Utilise Claude Haiku pour rester économique.
✨ L'IA génère un programme en JSON structuré, puis le serveur vérifie et corrige automatiquement les violations de règles (teacher absent de sa table, joueur sur deux tables, etc.) avec jusqu'à 2 itérations de correction.
Obtenir une clé API Anthropic
1
Créez un compte sur console.anthropic.com
2
Dans Settings → API Keys, créez une nouvelle clé
3
Ajoutez-la dans votre .env : ANTHROPIC_API_KEY=sk-ant-...
4
Redémarrez l'application — le bouton "Générer avec l'IA" apparaît dans l'onglet Programme
💸 L'usage de l'API Anthropic est payant. Une génération de programme coûte typiquement moins de 0.01 CHF en mode guidé (Haiku) et ~0.05 CHF en mode libre (Sonnet).

Gérer les séances

Depuis l'accueil, cliquez + Créer une nouvelle séance. Renseignez le nom, la date et créez des catégories de jeux. Les admins peuvent créer des séances privées visibles uniquement par les membres invités.

Les cartes de séances affichent des bandeaux d'alerte : 🔵 vote manquant ou incomplet · 🟠 nouveaux jeux proposés depuis votre dernier vote.

L'admin et le créateur de séance peuvent retirer un participant en cliquant le × à droite de son nom dans la barre des participants.

10
Génération IA ✨ IA
✨ Cette fonctionnalité nécessite une clé ANTHROPIC_API_KEY dans la configuration. Elle est absente de la version "sans IA".
Mode guidé — Claude Haiku

Renseignez les horaires, la pause déjeuner, le nombre de tables maximum et le nombre de créneaux tous ensemble vs en parallèle. L'IA génère un programme structuré rapidement et à moindre coût.

Mode libre — Claude Sonnet

Décrivez librement le programme en langage naturel. Plus puissant pour les contraintes complexes.

✍️ "2 créneaux tous ensemble le matin, 2 créneaux de 2 tables l'après-midi. Ramses ne joue pas à White Castle."
Processus de génération
1
Pré-calcul — le serveur calcule les contraintes (temps disponible, répartition idéale, teachers) et les injecte dans le prompt.
2
Génération — Claude produit un programme JSON structuré respectant les règles.
3
Vérification — le serveur vérifie les règles P1/P2. Si violations détectées, Claude reçoit les erreurs et corrige (max 2 itérations).
4
Post-traitement — le serveur injecte les teachers manquants, redistribue équitablement les joueurs et filtre les créneaux invalides.
📐
11
Règles du programme IA ✨ IA
P1
Le teacher est dans les joueurs de sa table — garanti par le serveur
P1
Un joueur ne peut être que sur une seule table par créneau — garanti par le serveur
P1
Les joueurs jouent idéalement au jeu qu'ils ont le plus voté
P2
Respecter exactement le nombre de créneaux tous ensemble / en parallèle demandé
P2
Respecter les contraintes min/max joueurs de chaque jeu
P2
Tous les participants jouent à chaque créneau
P2
Tables parallèles avec durées synchronisées et écart max 1 joueur
P3
Un joueur ne devrait pas être teacher sur plus de 3 jeux dans la journée
